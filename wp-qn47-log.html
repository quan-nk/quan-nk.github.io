1. Architecture & Core Concepts

2026-03-30

What is Quarto? — Quarto is an open-source publishing system. You write .qmd files (Markdown + YAML header), and Quarto renders them into HTML, PDF, etc. It's the successor to R Markdown, built by Posit (formerly RStudio).

The Pipeline

edit .qmd files
    ↓
quarto render          → builds HTML into docs/
    ↓
git add + commit + push  → pushes to GitHub
    ↓
GitHub Pages           → serves docs/ as a website

Key Files

_quarto.yml — The brain of your site. Controls:

  • project.type: website (vs book, blog)
  • project.output-dir: docs/ — where rendered HTML goes
  • project.render: glob patterns for which files to render (excludes wp-export/)
  • website.navbar: top navigation bar
  • website.sidebar: left sidebar — your site-wide table of contents
  • format.html.theme: visual theme (SCSS file or preset name)
  • format.html.toc: right-side "On this page" table of contents

theme-dark.scss — Custom dark theme. Uses SCSS (CSS superset with variables). Two sections:

  • /*-- scss:defaults --*/ — Bootstrap variables ($body-bg, $link-color, etc.)
  • /*-- scss:rules --*/ — Custom CSS rules for specific elements

Quarto uses Bootstrap 5 under the hood. Any Bootstrap variable can be overridden in the defaults section.

styles.css — Additional CSS overrides loaded AFTER the theme. Use this for fine-tuning (spacing, sizing) without touching the SCSS theme.

Each .qmd file has YAML frontmatter (between ---) controlling per-page settings:

---
title: "Post Title"
date: 2024-03-31
categories: ["Tag1", "Tag2"]
code-fold: true
---

Your markdown content here...
Three-layer layout: Your site uses the "documentation" pattern:
  • Top navbar — page-level navigation (Data Science, Mind Wandering, etc.)
  • Left sidebar — site-wide table of contents (all sections + all posts)
  • Right TOC — per-page headings ("On this page")
This is configured in _quarto.yml under website.sidebar (left) and format.html.toc (right).

2. How the Site Was Built (2026-03-30)

2026-03-30 08:00–10:00 ICT

Step 1: Scaffold

A Python script (~/.claude/skills/quarto-learning-site/scripts/scaffold.py) created ALL site files in one shot from a JSON config:

python3 scaffold.py site-config.json .

Created: _quarto.yml, index.qmd, about.qmd, resources.qmd, all data-science/ subsection directories, mind-wandering/ blog directory, .gitignore, styles.css, docs/.nojekyll.

Why a script? Token efficiency. One Bash call instead of 15+ Write calls.

Step 2: Git + GitHub

git init
git add -A
git commit -m "Initial Quarto site scaffold"
gh repo create wp-qn47 --public --source=. --push
gh api repos/quan-nk/wp-qn47/pages --method POST \
  -f "build_type=legacy" -f "source[branch]=main" -f "source[path]=/docs"

GitHub Pages configured to serve from docs/ on the main branch.

Step 3: WordPress Migration

Script: wp-export/migrate.py

  1. Reads wp-export/posts/posts-index.json for metadata (title, date, categories, status)
  2. Fetches content from WP REST API (public posts) or local cache (private posts fetched via WordPress MCP)
  3. Converts WordPress block HTML to Markdown (strips <!-- wp:* --> comments, converts HTML tags)
  4. Writes .qmd files with YAML frontmatter to mind-wandering/posts/ or mind-wandering/book-reviews/

Gotcha: Quarto reserves language: in YAML frontmatter for localization files. Renamed to lang-post: to avoid build errors.

Step 4: Render + Push Pattern

Every time you change content:

quarto render
touch docs/.nojekyll        # quarto render wipes docs/, restore this
git add <changed files> docs/
git commit -m "description"
git push

Why .nojekyll? GitHub Pages uses Jekyll by default, which ignores files starting with _. The empty .nojekyll file disables Jekyll processing.

3. Dark Theme & Layout Customization (2026-03-30)

2026-03-30 11:00 ICT

The Theme System

How Quarto themes work: Quarto uses Bootstrap 5 + Bootswatch. You can either:
  • Use a preset: theme: cosmo, theme: flatly, theme: darkly, etc.
  • Use a custom SCSS file: theme: theme-dark.scss
  • Layer both: theme: [darkly, custom.scss] (preset + overrides)

What was changed

Problem: "Posts > Posts" and "Book Reviews > Book Reviews" double nesting in sidebar.

Cause: Using contents: mind-wandering/posts (directory path) makes Quarto auto-create a section from the directory name. Wrapping it in section: "Posts" adds another layer.

Fix: Changed to glob pattern mind-wandering/posts/*.qmd which lists files directly without auto-creating a section header. Renamed outer section to "Essays" for clarity.

# BEFORE (double nesting)
- section: "Posts"
    contents: mind-wandering/posts

# AFTER (flat listing under "Essays")
- section: "Essays"
    contents:
      - mind-wandering/posts/*.qmd

Problem: "On this page" labels too close to vertical line.

Fix in styles.css:

nav#TOC .nav-link {
  margin-left: 0.6rem;
}

Dark theme (theme-dark.scss): Claude Docs-inspired colors.

$body-bg:      #1b1b1f;   // dark charcoal background
$body-color:   #c9c5bd;   // warm gray text
$link-color:   #d4956b;   // warm amber links
$headings-color: #e8e4dc; // cream headings
$navbar-bg:    #131316;   // darker navbar
$sidebar-bg:   #18181c;   // slightly lighter sidebar
$code-bg:      #222228;   // code block background
$border-color: #2d2d35;   // subtle borders

The /*-- scss:defaults --*/ section overrides Bootstrap variables. The /*-- scss:rules --*/ section adds custom CSS for specific elements (navbar border, sidebar hover color, TOC active state, etc.).

4. How to Customize From Here

Reference — updated 2026-03-30

Add a new tutorial page

  1. Create a .qmd file, e.g. data-science/statistics/descriptive.qmd
  2. Add YAML frontmatter: title, date, categories
  3. Add the file path to _quarto.yml sidebar under the right section
  4. quarto render && touch docs/.nojekyll && git add ... && git commit && git push

Add a new blog post

  1. Create mind-wandering/posts/YYYY-MM-DD-slug.qmd
  2. Add YAML frontmatter
  3. No need to edit _quarto.yml — the glob mind-wandering/posts/*.qmd auto-discovers new files
  4. Render + push

Change colors/fonts

Colors: Edit theme-dark.scss variables in the /*-- scss:defaults --*/ section.

Fonts: Add to theme-dark.scss:

$font-family-sans-serif: "Your Font", sans-serif;

To use Google Fonts, add to _quarto.yml:

format:
  html:
    mainfont: "Source Sans Pro"
    include-in-header:
      text: |
        <link href="https://fonts.googleapis.com/css2?family=Source+Sans+Pro&display=swap" rel="stylesheet">

Change the layout

  • Sidebar contents: Edit website.sidebar.contents in _quarto.yml
  • Navbar links: Edit website.navbar.left in _quarto.yml
  • Right TOC depth: Change toc-depth: 3 (shows h2+h3+h4 headings)
  • Sidebar collapse: collapse-level: 1 = all sections start collapsed. 2 = first level expanded.

Fine-tune spacing & sizing

Edit styles.css for quick CSS overrides (loaded after the theme). Use browser DevTools (F12) to inspect elements and find the right CSS selectors.

  • Navigation (sidebar, navbar)
  • Listings (blog pages)

5. Site Restructure (2026-03-30 ~12:00 ICT)

2026-03-30 12:00 ICT

What changed

Repo renamed: wp-qn47 → quan-nk.github.io

GitHub automatically serves repos named username.github.io at the root URL. No more /wp-qn47 path.

gh repo rename quan-nk.github.io --yes
git remote set-url origin https://github.com/quan-nk/quan-nk.github.io.git

New page structure:

quan-nk.github.io/
├── read/        ← essays + book reviews (was mind-wandering/)
│   ├── index.qmd    (listing page — auto-discovers all .qmd here)
│   └── *.qmd        (34 posts, flat — no subcategories)
├── learn/       ← data science (was data-science/)
│   ├── index.qmd    (overview: 4 pillars)
│   ├── measure/     (descriptive stats, distributions)
│   ├── compare/     (hypothesis testing, effect sizes)
│   ├── explain/     (causal inference, epi, regression)
│   └── predict/     (ML, classification, forecasting)
├── dissect/     ← NEW: paper walkthroughs
├── resources.qmd
├── about.qmd
└── index.qmd    (home)

Visual fixes:

  • Headings/titles: pale yellow #e8d5a0 — distinguishes from body text
  • Body text: warm white #e0ddd8 — brighter, more readable
  • Content width: body-width: 950px (was default ~750px). Set via format.html.grid in _quarto.yml
  • Sidebar nesting: fixed by using glob read/*.qmd instead of directory path read
Why globs vs directories in sidebar?
  • contents: mind-wandering/posts → Quarto auto-creates a section header from the directory name → double nesting
  • contents: [read/*.qmd] → lists files directly, no extra header → flat listing under your section name

Common tasks

Write a new essay

# 1. Create the file
touch read/2026-03-30-my-new-post.qmd

# 2. Add frontmatter + content
# (open in any text editor — VS Code, RStudio, vim, etc.)

---
title: "my post title"
date: 2026-03-30
categories: ["tag1", "tag2"]
---

Your content in markdown here...

# 3. Render + push
quarto render
touch docs/.nojekyll
git add read/2026-03-30-my-new-post.qmd docs/
git commit -m "Add post: my post title"
git push

No need to edit _quarto.yml — the glob read/*.qmd auto-discovers new files in the sidebar, and read/index.qmd listing auto-discovers them for the listing page.

Write a new tutorial (under learn)

# 1. Create the file inside the right subsection
touch learn/measure/descriptive-stats.qmd

# 2. Add frontmatter + content with R code

---
title: "descriptive statistics"
date: 2026-03-30
code-fold: true
---

```{r}
summary(mtcars)
```

# 3. Add to sidebar in _quarto.yml (tutorials need explicit listing)
# Under the "measure" section, add:
#   - learn/measure/descriptive-stats.qmd

# 4. Render + push (same as above)

Key difference from essays: tutorial pages must be added to _quarto.yml sidebar manually (under the right section). Essays auto-discover via glob.

Add a paper walkthrough (dissect)

# 1. Create the file
touch dissect/2026-03-30-some-paper.qmd

# 2. Write the walkthrough
# 3. Add to _quarto.yml sidebar under dissect section
# 4. Render + push

Change colors or fonts

Edit theme-dark.scss. Key variables at the top:

$body-bg:        #1b1b1f;   // page background
$body-color:     #e0ddd8;   // normal text color
$headings-color: #e8d5a0;   // h1/h2/h3 title color
$link-color:     #d4956b;   // link color
$sidebar-bg:     #18181c;   // sidebar background
$code-bg:        #222228;   // code block background
$border-color:   #2d2d35;   // all border lines

After editing, run quarto render to see changes.

Change content width

In _quarto.yml under format.html.grid:

grid:
  body-width: 950px     # main content area
  sidebar-width: 250px  # left sidebar
  margin-width: 200px   # right TOC area

Preview locally before pushing

quarto preview    # opens browser at localhost:port, auto-refreshes on save
# Ctrl+C to stop

This is the safest way to check changes. quarto render builds once; quarto preview watches for changes and rebuilds automatically.

The full publish cycle

# 1. Edit .qmd files (content) or .scss/.css (theme)
# 2. Preview locally
quarto preview

# 3. When satisfied, render the final build
quarto render

# 4. Restore .nojekyll (quarto wipes docs/ on render)
touch docs/.nojekyll

# 5. Stage, commit, push
git add <your-changed-files> docs/
git commit -m "description of change"
git push

# 6. Wait 1-2 minutes for GitHub Pages to update
# 7. Hard refresh browser (Cmd+Shift+R) to see changes
